Guia rápido — Como usar o SDD

O fluxo em 5 passos
  1. 1
    /sdd:spec-init "descrição"
    Cria a pasta da feature com spec.json
  2. 2
    /sdd:spec-requirements {feature}
    IA gera o QUE o sistema deve fazer → você revisa
  3. 3
    /sdd:spec-design {feature}
    IA gera o COMO vai ser feito → você revisa
  4. 4
    /sdd:spec-tasks {feature}
    IA gera as tarefas de implementação → você revisa
  5. 5
    /sdd:spec-impl {feature}
    Agora sim: IA implementa o código aprovado
Por que usar SDD?
Rastreabilidade
Cada linha de código tem um requisito associado. Você sempre sabe por que algo foi feito.
Controle humano
A IA propõe; você aprova. Nenhuma fase avança sem sua revisão explícita.
AI-agnóstico
O mesmo workflow funciona com Claude Code ou Gemini. Você não fica preso a uma ferramenta.
