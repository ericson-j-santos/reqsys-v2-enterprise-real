# Fronteira de autonomia — Autonomous Delivery Cycle

## Autorizado

- Avaliar PRs candidatos.
- Validar labels.
- Validar workflows obrigatórios.
- Fazer squash merge se todos os critérios passarem.
- Observar CI pós-merge.
- Registrar próximos incrementos em fila report-only.

## Não autorizado

- Criar código novo a partir da fila sem novo contexto.
- Corrigir CI vermelho automaticamente.
- Aplicar labels sozinho.
- Forçar merge.
- Ignorar revisão humana em PR crítico.
- Alterar branch protection.

## Princípio

Autonomia para operar entrega verde e autorizada; não autonomia para decidir escopo novo sem validação contextual.
