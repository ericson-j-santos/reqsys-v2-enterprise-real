Como usar — Guia de Comandos

Workflow passo a passo

Phase 0 — Steering

Configura o contexto global do projeto. Execute uma vez.
/sdd:steering

1a — Inicializar spec

Cria a estrutura da spec a partir de uma descrição.
/sdd:spec-init "descrição"

1b — Gerar Requisitos

IA gera o O QUÊ o sistema deve fazer. Você revisa.
/sdd:spec-requirements {feature}

⛔ Revisão Humana

Leia o requirements.md gerado. Se OK, avance. Se não, peça ajustes.

1c — Gerar Design

IA gera o COMO será implementado. Você revisa.
/sdd:spec-design {feature}

1d — Gerar Tarefas

IA divide o design em tarefas de 1-3h.
/sdd:spec-tasks {feature}

2 — Implementar

IA escreve o código. Execute por tarefa.
/sdd:spec-impl {feature} 1.1

Fast-track com -y

Pula a pausa de aprovação naquela fase. Use só quando tiver certeza.
/sdd:spec-design {feature} -y

Status a qualquer hora

Mostra fase atual, aprovações e próximos passos.
/sdd:spec-status {feature}
