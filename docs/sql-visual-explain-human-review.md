# Revisão Humana — SQL Visual Explain Stack

## Pontos para revisar no PR

- Nomenclatura dos arquivos.
- Localização dos artefatos em `docs/`, `examples/`, `scripts/`, `tests/` e `public/`.
- Escopo do parser heurístico.
- Se o HTML deve ser mantido em `public/` ou migrado para documentação estática.
- Se SQLGlot entra no próximo PR ou no mesmo ciclo após CI verde.

## Critério para aprovação

O PR pode avançar quando:

1. CI estiver verde.
2. Script e teste forem aceitos.
3. Não houver exposição de dado sensível.
4. A estratégia de próximo incremento estiver clara.
