# CI Incident Intelligence

## Objetivo

Adicionar inteligência operacional para classificação e análise de incidentes de CI/CD.

## Capacidades iniciais

- Detecta workflows concluídos.
- Classifica incidentes básicos.
- Sugere ações operacionais.
- Correlaciona workflow, branch e commit.
- Publica resumo operacional.
- Publica artifact reutilizável.

## Classificações atuais

- Regressão genérica.
- Possível falha flaky.
- Regressão de cobertura/testes.
- Regressão de observabilidade operacional.

## Benefícios

- Redução de MTTR.
- Menos investigação manual.
- Menos reruns desnecessários.
- Base para auto-remediação futura.
- Maior maturidade operacional.

## Próximos incrementos previstos

- Analytics históricos.
- Detecção estatística flaky.
- Correlação PR ↔ deploy ↔ incidente.
- Score de estabilidade operacional.
- Auto-remediação governada.
